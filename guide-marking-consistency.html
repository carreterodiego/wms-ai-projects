Marking Consistency GPT

User Guide for Testing

⚠ Important — this is a prototype
This tool is in active development and has not yet been independently validated. Always apply your own professional judgment. Your feedback is what will improve it.

What this tool does

The Marking Consistency GPT applies your rubric to a student submission and returns a structured grade recommendation with student-facing feedback. It is designed to support consistent marking across a cohort — applying the same rubric logic to every submission regardless of where it falls in your marking queue.

Every recommendation must be reviewed and approved by you before any grade is finalised. Nothing produced by this tool should be communicated to a student without your review.

Before you start

Have the following ready:

  • The assignment rubric. With band descriptors if possible — the more detailed, the better the output.
  • The assignment instructions. The brief or task description students received.
  • A student submission to test with. One is enough for a first test. You can work through more if you want.
  • Your own grade for that submission. So you can compare the tool's recommendation against your judgment.
🔒 Privacy — student data
DO NOT include any student-identifying information. Remove or replace all names, student ID numbers, and any other details that could identify a student before pasting content into this tool. You are responsible for managing your own workflow to keep track of which anonymised submission belongs to which student.

How it works — step by step

  1. Open the tool and answer the opening questions. It will ask for the assignment name, cohort level, your name, whether this is a final or early assessment, any calibration instructions (e.g. default to mid-band unless clearly exceptional), your preferred feedback tone, and preferred feedback length.
  2. Upload the rubric and assignment instructions. Both are required. Upload as Word documents, PDFs, or paste as plain text. The tool reads and extracts the band descriptors directly — you do not need to describe them.
  3. Confirm the rubric has been read correctly. The tool will summarise what it has extracted. If it has misread something, correct it before proceeding.
  4. Paste or upload the student submission. The tool will apply the rubric and return a grade recommendation with justification and student-facing feedback.
  5. Review the recommendation carefully. Compare it to your own judgment. Note where it agrees and where it diverges. If you disagree, trust your judgment — you are the marker.
  6. Sign off or adjust. The tool requires acknowledgment before any output is used. You remain responsible for every grade.
🔒 Privacy — student data
DO NOT include any student-identifying information. Remove or replace all names, student ID numbers, and any other details that could identify a student before pasting content into this tool. You are responsible for managing your own workflow to keep track of which anonymised submission belongs to which student.
If the tool issues a rubric quality warning, it means the rubric lacked detail in some areas. You can improve the rubric and retry, or proceed — but note that the recommendation may be less reliable.

What to watch for

  • Does the band recommendation match your own judgment? If not, why?
  • Is the justification mapped to specific rubric criteria, or is it generic?
  • Does the student-facing feedback feel appropriate for the cohort and assignment?
  • Did the tool flag any academic integrity signals? If so, do you agree it warranted attention?
  • Did the tool reward length or fluency over analytical quality?

How to give feedback on the tool

After testing, please note:

  • How closely the recommendation matched your own grade
  • Where the justification was strong or weak
  • Any output that felt misleading or incorrect
  • Whether the process felt usable within a normal marking workflow
  • How long it took compared to marking without the tool
