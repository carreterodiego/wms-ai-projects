Assessment projects

Marking Consistency GPT
Prototype — open for testing ↗ User guide Prompt + KB

Applies a rubric to student submissions and returns a structured grade recommendation with student-facing feedback. Designed to support consistent marking across a cohort — the same rubric logic applied to every submission, regardless of where it falls in the marking queue. The marker reviews and signs off every recommendation before it reaches a student.

  • Key feature: 9-question interview at session start locks in context before any submission is accepted
  • Key feature: marker uploads the actual rubric and assignment instructions — agent reads and extracts the band descriptors directly
  • Key feature: rubric quality gate — warns the marker if the rubric is structurally weak before marking begins
  • Key feature: anti-bias constraint — agent assesses analytical quality, not length or writing style
  • Key constraint: mandatory marker sign-off on every output
Feedback Enhancer GPT
Prototype — open for testing ↗ User guide Prompt + KB

Takes a marker's shorthand notes and turns them into polished, rubric-aligned feedback ready to share with a student. The marker makes every evaluative decision — the agent handles the writing. Designed to save significant time at the feedback-writing stage without reducing the quality or personalisation of what the student receives.

  • Key feature: Invention Guard — hard constraint preventing the agent from adding anything not explicitly in the marker's notes
  • Key feature: if notes are too sparse for a criterion, the agent flags the gap rather than filling it with generic content
  • Key feature: tone calibration — marker sets their preferred feedback style at session start; the agent applies it silently throughout

Research projects

Ethics Application Preparation Assistant
Prototype — open for testing ↗ User guide Prompt only

Guides WMS staff and students through the preparation stage before submitting a research ethics application. Identifies the correct form and the documents required. Does not approve research, complete forms, or submit applications — it handles the navigation, not the content.

  • Key feature: gate-based decision logic — one question at a time, pathway determined by the user's answers
  • Design note: prompt-only because the logic is procedural — no rubric depth needed, fits within the instruction limit without compromise

Teaching support projects

WMS Case Builder
Prototype — open for testing ↗ User guide Prompt + KB

Helps lecturers and tutors generate high-quality fictional teaching cases for classroom discussion, assignments, workshops, and exams. Guides the user through a structured interview, generates three distinct scenario options, then expands the chosen one into a full case. Runs a quality check before every output to make sure the case presents a genuine dilemma with no obvious answer.

  • 3 case styles: Highly Structured (exams), Balanced (coursework), Story-Driven (dilemma must be inferred)
  • Optional: generates a teaching note for the lecturer including discussion questions and common student errors
  • Optional: lecturer uploads a reference case — agent matches its tone and style

Evidence base

The WMS AI agents were built alongside a review of peer-reviewed literature on AI-assisted grading, feedback quality, and GenAI in higher education. Every design choice is traceable to a finding or a known limitation in that literature.

  • AI as support, not replacement — the consistent finding across the literature; all agents are built on this principle (Deepshikha, 2025; Deepshikha, 2026; Implementation Considerations, 2025)
  • Rubric quality drives output reliability — detailed band descriptors in every knowledge base; rubric quality gate in the Marking Consistency agent (Huang et al., 2026)
  • Human marker inconsistency is documented and significant — even with a shared rubric, graders frequently disagree; inter-rater reliability among humans can be very low (Messer et al., 2025; Wetzler et al., 2025)
  • Fabrication risk — LLMs can add unreferenced content; addressed directly by the Invention Guard (Huang et al., 2026)
  • Length and fluency bias — LLMs over-reward well-written submissions regardless of analytical depth; explicitly countered in the Marking Consistency prompt (AlGhamdi et al., 2026; Trikoili et al., 2024)
  • Feedback quality matters pedagogically — narrative feedback is valued more than numeric scores; vague or grade-justifying feedback has limited learning impact (Implementation Considerations, 2025)
  • Honest gap — no peer-reviewed literature exists specifically on AI grading of business school case analysis; the assessment agents are framed as exploratory in that space

External tools assessed

Tools assessed for potential relevance to WMS teaching, assessment, or research workflows. Know a tool we should look at? Submit a suggestion on the feedback page.

Score Automator
Excel — no AI

A rubric calculator built in Excel. Takes a marker's score inputs, applies weightings, calculates the final grade, and assembles pre-written feedback templates by grade tier.

  • What it is: a local Excel tool for grade arithmetic and feedback template assembly — no AI component, no external connectivity
  • Where it adds value: structured record-keeping and arithmetic automation — useful for markers who want workflow structure without AI involvement
  • Key limitations: it is a field-filling tool, not a consistency tool — two markers using it on the same submission produce the same arithmetic, not necessarily the same grade; feedback is identical for every student at a given grade tier with no individual developmental value
  • Data & governance: N/A — local Excel file; no data leaves the institution
  • Relationship to WMS AI projects: complementary, not competing — addresses a different problem at a different point in the workflow
ResearchBuddyAI
Not recommended Web-based SaaS

Markets itself as a multi-model AI research assistant that cross-validates outputs to reduce hallucinations. Assessed following a LinkedIn post that came to WMS attention. Assessment outcome: not suitable for institutional use.

  • What it is: appears to be primarily a local academic networking and supervision platform — LinkedIn activity references virtual research labs, a specific regional academic community, and internship roles at approximately NZ$60/month for subscriber outreach via WhatsApp; the AI research tool product appears to have been added on top of this base and marketed to a Western audience using AI-generated assets
  • Where it adds value: N/A — no credible value case identified after assessment
  • Key limitations: the core claim (multi-model cross-validation producing zero hallucinations) is architecturally plausible as a concept but entirely unsupported — no peer-reviewed evidence, no methodology documentation, no independent validation; the LinkedIn post used what appears to be a fake AI-generated UI screenshot; the product website has no named team, no founding story, and emoji as section headers throughout; monetisation relies on opaque credit consumption, fake urgency timers, and comparison tables claiming 100% savings versus human researchers
  • Data & governance: non-starter institutionally — their own privacy policy confirms work is processed through third-party infrastructure with no institutional data agreement; data leaves the university with no contractual protection
  • Relationship to WMS AI projects: none — nothing here to build on or reference

How the agents are built

Two-component design

Most agents use a two-component architecture: a system prompt that defines the agent's logic and behaviour, and a separate knowledge base document that carries depth — rubric standards, research grounding, quality criteria.

The two components update independently. Changing the rubric or research base does not require rewriting the agent logic. The same agent can be deployed to a new course without rebuilding it.

Knowledge base governance

The knowledge base is a locked reference document the agent reads silently. Individual markers and users never see or access it. It contains research grounding and quality standards that apply across all courses.

It is maintained by the toolkit owner only. Assignment-specific content — rubrics, instructions — is provided by the user at session start via document upload, not baked into the KB.
